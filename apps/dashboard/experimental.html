Legacy voice WebUI

The original qwen3 WebUI is a separate project. Maya Unified does not embed or proxy it — that would attach a second UI to the same voice stack and waste VRAM.

Important

Do not run both servers at once unless you intend to load models twice. Use Maya Unified (launch.py on port 8090) or the standalone qwen3 server — not both for the same session.

Duplex Voice (recommended)

Browser mic duplex stack from Duplex Voice Lab — Qwen3-ASR → LM Studio → faster-qwen3-tts, with generation-ID barge-in. Open /duplex on this gateway (port 8090).

Open Duplex Voice 
# Terminal 1 — ASR
.\scripts\start-asr.ps1
# Terminal 2 — gateway (LM Studio on :1234)
python launch.py

Run the standalone WebUI

From the repo root, in a separate terminal:

cd packages/voice-runtime
.venv\Scripts\python.exe server.py

Default URL: http://localhost:7861

Open qwen3 UI (7861) Back to Unified Dashboard

What lives where

  • Maya Unified — dashboard, settings, memory/tools/EQ panels, one voice agent instance via /api/voice/agent/*.
  • qwen3-voice-agent — read-only upstream; run server.py only when you want the legacy full WebUI in isolation.
  • Shared data — agent memory/DBs are copied into maya-unified/data/ on first start; standalone qwen3 still uses its own qwen3-voice-agent/data/ unless you sync manually.
